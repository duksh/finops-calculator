# GitHub Product Ops Setup for FinOps Calculator Roadmap

This guide sets up execution backlog in `duksh/finops-calculator`.

## 1) Milestones

Create these milestones:

- `2026-Q1 Foundation and Trust`
- `2026-Q2 ITAM and Portfolio Convergence`
- `2026-Q3 Governance Expansion`
- `2026-Q4 Decision Engine and Enterprise`

## 2) Labels

### Work item type
- `type:epic`
- `type:story`
- `type:task`
- `type:kpi`
- `type:feedback`

### Domain
- `domain:finops`
- `domain:itam`
- `domain:greenops`
- `domain:ai-governance`

### Priority
- `priority:p0`
- `priority:p1`
- `priority:p2`

### Status
- `status:blocked`
- `status:needs-research`
- `status:ready`

## 3) GitHub Project v2 fields

Recommended custom fields:

- `Quarter` (Q1, Q2, Q3, Q4)
- `Type` (Epic, Story, Task, KPI)
- `Domain` (FinOps, ITAM, GreenOps, AI Governance)
- `KPI Link` (text/url)
- `Target Date`
- `Confidence` (Low, Medium, High)

## 4) Issue naming conventions

- Epic: `[EPIC][QX] <epic title>`
- Story: `[STORY][QX] <story title>`
- KPI: `[KPI][QX] <metric name>`
- Feedback: `[FEEDBACK] <summary>`

## 5) Feedback workflow

1. Capture in Discussions or feedback issue.
2. Triage weekly.
3. Convert to story and link to epic.
4. Prioritize into quarter milestone.

## 6) Cadence

- Weekly: feedback triage and backlog grooming.
- Bi-weekly: sprint planning and issue sizing.
- Monthly: KPI check-in.
- Quarterly: milestone closure and release summary.
